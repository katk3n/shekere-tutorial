## 0. シェーダーアートとは

### シェーダーアートの定義
**シェーダーアート**とは、GPU（Graphics Processing Unit）で実行されるシェーダープログラムを使って作成される芸術作品です。数学的な計算により、美しいビジュアルエフェクトやパターンをリアルタイムで生成します。

### シェーダーアートの特徴
- **リアルタイム生成**: 毎フレーム計算によって映像を生成
- **数学的美しさ**: 数式や関数による規則的で美しいパターン
- **インタラクティブ**: 音楽、マウス、MIDIなどとの連携が可能
- **高いパフォーマンス**: GPUの並列処理により高速動作

### シェーダーアートの応用分野
- **VJパフォーマンス**: ライブ音楽イベントでの映像演出
- **インスタレーション**: 美術館やギャラリーでの展示作品
- **デジタルアート**: オンラインでの作品公開・共有
- **ゲーム開発**: 特殊効果やポストプロセッシング

## 0.1 shekereフレームワークとは

### shekereの概要
**shekere**は、シェーダーアート作成に特化したフレームワークです。WGSL（WebGPU Shading Language）を使用し、音響連携やリアルタイム制御に優れた機能を提供します。

### shekereの主要機能

#### 1. シェーダー関連
- **WGSL サポート**: 最新のWebGPU技術による高性能描画
- **Hot Reload**: コード変更の即座反映でライブコーディング体験
- **マルチパス レンダリング**: 複数段階の処理による複雑なエフェクト

#### 2. 音響連携
- **リアルタイム スペクトラム解析**: 音楽の周波数成分をビジュアル化
- **MIDI制御**: 楽器やコントローラーからのリアルタイム操作
- **OSC連携**: TidalCyclesなど外部ソフトウェアとの連携

#### 3. インタラクション
- **マウス入力**: カーソル位置による効果制御
- **時間制御**: 経過時間による自動アニメーション
- **パラメータ調整**: 設定ファイルによる細かい制御

### shekereの利点
- **学習しやすさ**: 最小限の設定でシェーダーアートを開始
- **拡張性**: 単純なパターンから複雑な作品まで対応
- **パフォーマンス**: WebGPUによる高速描画
- **コミュニティ**: 豊富なサンプルコードと活発な開発

```

├── config.toml      # 設定ファイル

```

### 実行方法
```bash
shekere config.toml
```

## 2. WGSLの基礎知識

### fragment shaderとは
fragment shader（フラグメントシェーダー）は、画面の各ピクセルの色を決定するプログラムです。GPUが並列実行し、リアルタイムで美しいグラフィックスを生成します。

### 基本的なWGSL構文

#### データ型
```wgsl
// スカラー型
var time: f32 = 1.0;        // 32bit浮動小数点
var count: u32 = 10u;       // 32bit符号なし整数
var flag: bool = true;      // 真偽値

// ベクトル型
var position: vec2<f32> = vec2(0.5, 0.5);  // 2Dベクトル
var color: vec3<f32> = vec3(1.0, 0.0, 0.0); // 3Dベクトル（RGB）
var rgba: vec4<f32> = vec4(1.0, 0.0, 0.0, 1.0); // 4Dベクトル（RGBA）
```

#### 関数の基本構造
```wgsl
@fragment
fn fs_main(in: VertexOutput) -> @location(0) vec4<f32> {
    // ここにシェーダーのロジックを書く
    return vec4(1.0, 0.0, 0.0, 1.0); // 赤色を返す
}
```

## 3. shekereの基本関数

### shekereフレームワーク専用ヘルパー関数

shekereフレームワークには、シェーダーアート制作を簡単にするための**専用ヘルパー関数**が用意されています。これらの関数は標準のWGSLには含まれておらず、shekereフレームワークが独自に提供する機能です。

---

### NormalizedCoords() - 正規化座標変換
**機能**: 画面座標を正規化座標（-1.0〜1.0）に変換します。

**用途**: 
- 画面の中心を原点(0,0)とした座標系での作業
- 画面サイズに依存しない描画処理
- 対称的なパターン作成

**座標系の説明**:
```wgsl
let uv = NormalizedCoords(in.position.xy);
// uv.x: 左端=-1.0, 中央=0.0, 右端=1.0
// uv.y: 下端=-1.0, 中央=0.0, 上端=1.0
```

**実践例**:
```wgsl
@fragment
fn fs_main(in: VertexOutput) -> @location(0) vec4<f32> {
    let uv = NormalizedCoords(in.position.xy);
    
    // 中心からの距離を計算
    let distance = length(uv);
    
    // 距離に応じた色を生成
    let color = vec3(distance, 0.0, 1.0 - distance);
    return vec4(ToLinearRgb(color), 1.0);
}
```

**注意事項**:
- この関数はshekereフレームワーク専用です
- 標準のWGSLやその他のフレームワークでは使用できません
- 画面の縦横比（アスペクト比）は自動的に考慮されます

---

### ToLinearRgb() - 色空間変換
**機能**: sRGB色空間からリニア色空間に変換します。

**必要性**: 
- 数学的に正確な色の計算を行うため
- 色の加算・減算・乗算を正しく実行するため
- 最終的な画面表示で正しい色を得るため

**使用方法**:
```wgsl
let color = vec3(1.0, 0.5, 0.0); // オレンジ色（sRGB）
return vec4(ToLinearRgb(color), 1.0); // リニア色空間に変換
```

**色空間の違い**:
- **sRGB**: 人間の視覚特性に合わせた色空間（通常の画像ファイル）
- **リニア**: 物理的に正確な光の強度を表現する色空間（GPU計算に最適）

**実践例**:
```wgsl
@fragment
fn fs_main(in: VertexOutput) -> @location(0) vec4<f32> {
    let uv = NormalizedCoords(in.position.xy);
    
    // 複数の色を数学的に混合
    let color1 = vec3(1.0, 0.0, 0.0); // 赤
    let color2 = vec3(0.0, 1.0, 0.0); // 緑
    let mixFactor = (uv.x + 1.0) * 0.5; // -1〜1を0〜1に変換
    
    // 正しい色空間で混合計算
    let mixedColor = mix(color1, color2, mixFactor);
    
    // 必ずToLinearRgb()を通してリニア色空間に変換
    return vec4(ToLinearRgb(mixedColor), 1.0);
}
```

**重要な注意**:
- **すべてのfragment shaderの最終出力**でToLinearRgb()を使用してください
- この関数もshekereフレームワーク専用です
- 色の計算を正確に行いたい場合は必須の関数です

---

## 4. 色の理論

### RGB色空間
- **R (Red)**: 赤成分 (0.0〜1.0)
- **G (Green)**: 緑成分 (0.0〜1.0)
- **B (Blue)**: 青成分 (0.0〜1.0)
- **A (Alpha)**: 透明度 (0.0〜1.0)

### 基本色の表現
```wgsl
vec3(1.0, 0.0, 0.0)  // 赤
vec3(0.0, 1.0, 0.0)  // 緑
vec3(0.0, 0.0, 1.0)  // 青
vec3(1.0, 1.0, 1.0)  // 白
vec3(0.0, 0.0, 0.0)  // 黒
vec3(1.0, 1.0, 0.0)  // 黄
vec3(1.0, 0.0, 1.0)  // マゼンタ
vec3(0.0, 1.0, 1.0)  // シアン
```

## 5. 実践例解説

### 例1: 単色表示
最もシンプルなシェーダーです。画面全体を指定した色で塗りつぶします。

```wgsl
@fragment
fn fs_main(in: VertexOutput) -> @location(0) vec4<f32> {
    let color = vec3(1.0, 0.0, 0.0); // 赤色
    return vec4(ToLinearRgb(color), 1.0);
}
```

### 例2: UV座標によるグラデーション
座標値を色に変換することで、グラデーション効果を作ります。

```wgsl
@fragment
fn fs_main(in: VertexOutput) -> @location(0) vec4<f32> {
    let uv = NormalizedCoords(in.position.xy);
    
    // X座標を赤成分に、Y座標を緑成分に
    let color = vec3(
        (uv.x + 1.0) * 0.5,  // -1〜1 を 0〜1 に変換
        (uv.y + 1.0) * 0.5,  // -1〜1 を 0〜1 に変換
        0.0                   // 青成分は0
    );
    
    return vec4(ToLinearRgb(color), 1.0);
}
```

### UV座標とは何か

#### UV座標の基本概念
**UV座標**とは、2D画面上の位置を表現するための座標系です。3D空間の座標がX、Y、Zで表現されるのに対し、2D画面ではU、Vの2つの軸で位置を表現します。

#### なぜ「UV」と呼ばれるのか
- **U軸**: 水平方向（横方向）の位置
- **V軸**: 垂直方向（縦方向）の位置
- X、Y、Zが3D空間で使われるため、2D画面座標では異なる文字（U、V）を使用します

#### シェーダーでのUV座標の役割
```wgsl
// 変数名「uv」は「UV座標」から来ている
let uv = NormalizedCoords(in.position.xy);
```

UV座標は以下の用途で使用されます：
- **位置の特定**: 画面上の任意の点を数値で表現
- **パターン生成**: 座標値を使って数学的なパターンを作成
- **テクスチャマッピング**: 画像を画面に貼り付ける際の位置指定
- **グラデーション**: 位置に応じた色の変化を作成

#### 座標系の種類
シェーダーでは主に2つの座標系が使用されます：

1. **正規化座標（NormalizedCoords）**:
   - 範囲: -1.0 〜 1.0
   - 中心: (0.0, 0.0)
   - 用途: 対称的なパターン、中心基準の計算

2. **テクスチャ座標**:
   - 範囲: 0.0 〜 1.0
   - 左上: (0.0, 0.0)、右下: (1.0, 1.0)
   - 用途: 画像の貼り付け、左上基準の計算

#### UV座標の実用例
```wgsl
let uv = NormalizedCoords(in.position.xy);

// 中心からの距離を計算
let distance = length(uv);

// 角度を計算
let angle = atan2(uv.y, uv.x);

// 格子パターンの生成
let grid = fract(uv * 10.0);
```

### 例3: UV座標の可視化
座標系の理解を深めるため、UV座標そのものを色として表示します。この例では、座標値を直接色に変換することで、画面上の各点がどのような座標値を持っているかを視覚的に確認できます。

```wgsl
@fragment
fn fs_main(in: VertexOutput) -> @location(0) vec4<f32> {
    let uv = NormalizedCoords(in.position.xy);
    
    // 座標値を可視化（負の値も表示するため絶対値を使用）
    let color = vec3(
        abs(uv.x),  // X座標の絶対値を赤成分に
        abs(uv.y),  // Y座標の絶対値を緑成分に
        0.5         // 青成分は固定値
    );
    
    return vec4(ToLinearRgb(color), 1.0);
}
```

**この例の解説**:
- **赤成分**: X軸の位置（左端=0.0、中央=0.0、右端=1.0）
- **緑成分**: Y軸の位置（下端=0.0、中央=0.0、上端=1.0）
- **青成分**: 固定値0.5（色を見やすくするため）
- **abs()関数**: 負の値を正の値に変換（-1.0〜1.0を0.0〜1.0に）

**期待される結果**:
- 画面の中心が最も暗く（両軸とも0.0）
- 画面の角が最も明るく（両軸とも1.0）
- 横方向の変化が赤色で、縦方向の変化が緑色で表示される

## 6. 学習のポイント

### デバッグのコツ
1. **色で値を可視化**: 数値を色として表示することで、計算結果を視覚的に確認できます
2. **段階的な変更**: 一度に多くを変更せず、小さな変更を積み重ねましょう
3. **Hot Reload活用**: shekereのホットリロード機能で、リアルタイムに結果を確認できます

### よくある間違い
1. **座標系の混同**: NormalizedCoords()は-1〜1、tex_coordsは0〜1の範囲です
2. **色空間の忘却**: ToLinearRgb()を忘れると色が正しく表示されません
3. **型の不一致**: f32とu32、vec2とvec3など、型の違いに注意しましょう
